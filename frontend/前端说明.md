# 前端说明

本文是 **Archigraph（档图）** 前端的**总说明**：怎么启动、代码怎么组织、设计怎么实现。
读者对象：接手维护、做评审、或想理解"为什么这么写"的人。读完应能独立改动任一部分。

> 项目名 Archigraph = archive（档案）+ graph（图谱），意为「把知识像档案归档成图谱」，
> 与招牌组件 AgentRoom 的「像素档案员」隐喻一脉相承。

---

## 1. 快速启动

    cd frontend
    cp .env.example .env     # 按需修改 VITE_API_BASE_URL（默认指向本地后端）
    npm install
    npm run dev              # 开发服务器，默认 http://localhost:5173

**联调前提**：前端只接真实后端才有意义（动画状态来自真实 RunEvent）。需要：
- 后端 `uvicorn` 起着（默认 :8000）
- Neo4j 起着（Docker Compose，共享库）
- `.env` 配好真实 LLM key

**开发预览**：`npm run dev` 后访问 `/?preview` 进 StyleGallery，用真实组件并排渲染 AgentRoom
全部 12 状态，一眼看全。`App.tsx` 里 `showGallery` 仅在 `import.meta.env.DEV` + 带 `?preview` 时挂载，不影响生产路由。

## 3. 它是什么

**Archigraph（档图）**——一个知识图谱 GraphRAG 工作台的前端：把"文档入库 → 实体关系抽取 →
图谱构建 → 向量+图谱检索 → 带引用回答"这条链路，用三个视图可视化出来，并配一个**像素小人
AgentRoom**实时反映后端在做什么。问答视图支持**多会话多轮对话**（见 §7.1）。

- 技术栈：React 19 + Vite 8 + TypeScript，图谱可视化用 Cytoscape.js
- 定位：**工作台是主体，AgentRoom 是配角**——动画只为反映程序正在执行什么，让前端丰富
- 风格：冷调 slate 中性 + 靛紫强调（对标 Linear/Notion 的工程感，刻意避开暖灰）

## 5. 设计系统（tokens.css）

所有颜色、间距、字阶、阴影都抽成 CSS 变量，集中在一处。这是"精致度"的根基——
组件里不写魔法数字，全用 token，保证全局一致。

- **表面层级**：`--color-bg`(页面底) → `--color-surface`(卡片) → `--color-surface-sunken`(下沉槽)
- **文字四档**：`--color-text` → `-body` → `-muted` → `-subtle`，从强到弱
- **强调色靛紫**：`--color-accent` 一族（hover/active/soft/border），给足交互层次
- **AgentRoom 房间色**：单独一套 `--room-*`（深紫房间）+ `--dude-*`（多彩小人）
- **签名字体**：等宽字体 `--font-mono` 承载"可溯源数据"（chunk ID、计数、时间戳），
  呼应引用可追溯护城河

## 6. 数据流（单向，红线）

```
后端 SSE ──RunEvent──> useRunEvents ──currentStage──> AgentRoom
                    └──events[]──> AgentRoom 内部的 RunEventTimeline（画布下方运行轨迹）
```

**红线**：AgentRoom 的 `stage` **只能来自真实 RunEvent**，前端绝不伪造状态。
`useRunEvents(runId)` 订阅 SSE，累积事件并派生 `currentStage`；没后端时永远 `idle`。
这条单向流是整个动画可信度的基础，加任何状态都不能绕过它。

后端契约锁定：`Stage` 12 枚举 + 终态事件携带 `answer`。后端注释明确"改名会破坏前端状态机"，
前端这边 `types/runEvent.ts` 与之同源。**升级 Agentic RAG 后契约未变**，前端无需改动。

## 7. 三个视图

### 问答工作台（WorkbenchView，主视图）
**三列布局**：`ConversationSidebar`(会话列表) | `ChatThread`+`ChatComposer`(对话区) |
`AgentRoom`(含运行轨迹) + `CitationPanel`(引用证据)。
> 引用面板原在中列（挤占对话区），已挪到右列；对话区因此更大。AgentRoom 下方原显示
> 状态文字（「待命/抽取」），已换成 `RunEventTimeline`（运行轨迹），状态信息靠像素小人
> 动作+家具表达即可。

**多会话多轮对话**（核心能力）：
- 左侧会话侧边栏：`GET /api/conversations` 列表，新建/切换/删除（删除带二次确认对话框，键盘可达）。
- 新建会话 → `POST /api/conversations` 拿 id → 设为当前 → 清空 messages → 侧边栏 unshift。
- 提问 → `POST /api/chat` 带 `conversationId`（首问 null，追问回传响应值）→ 拿 `runId` 订阅 SSE
  → 终态落答案消息。**终态只清 `chatRunId` 解除 busy，不清 `conversationId`**（追问不断的关键）。
- 切换会话 → `GET /api/conversations/{id}` 拉历史 → `ConversationMessage` 转 `ChatMessage` 回灌
  （复用现有渲染，引用角标/CitationPanel 零改动）。
- 引用面板展示**最近一条** agent 回答的引用，契合"当前回答"语义。
- **切换视图不丢会话**：三个视图（工作台/文档库/图谱）在 `App.tsx` 用 `hidden` 属性切显隐
  而非条件渲染，组件不卸载、state 天然保留——切到文档库再切回来，当前会话和对话历史仍在。

> **直连真实后端**：会话 CRUD 走 `api/conversations.ts` 调真实 `/api/conversations/*`，
> 提问走 `POST /api/chat` 带 `conversationId`。开发期曾用 `USE_MOCK` 开关 + 内存 mock
> 数据独立打通 UI 流，后端就绪后已切真实并删除整段 mock 数据（见 §10 切真实步骤留作参考）。

### 文档库（LibraryView）
`GET /api/documents` 列表；上传走原生 `fetch + FormData`（绕开 apiFetch 的 JSON content-type）
→ 起 Run → SSE 订阅 → 终态刷新列表。删除走异步 Run，**点删除先弹二次确认 modal**（与 ConversationSidebar
风格一致），确认后才带 `?confirm=true` 调 `DELETE /api/documents/{id}`（后端安全设计要求显式确认）。

### 图谱探索（GraphView）
接 `GET /api/graph/*`。**前后端字段名不一致**（后端 `name/type`，前端 `label/entityType`），
在 `api/graph.ts` 收口一层映射，不改后端。Cytoscape.js 渲染，支持实体邻域展开与模糊搜索。

## 8. AgentRoom（像素小人房间）—— 本前端招牌

### 8.1 它解决什么 / 定位
让"后端正在执行什么"变得肉眼可见。小人按状态横向飘到对应家具前，家具自己运转表达动作。
**不是装饰**，是程序执行状态的实时可视化（尤其 Agentic RAG 多轮检索时，小人在工位间
反复穿梭能直观体现 Agent 的自主决策）。

### 8.2 演进过程（为什么是现在这样）

经历了三轮迭代，每轮都是"发现上一版的根本问题、对症重构"：

**第一版 PixelAgent（已废弃）**：精细分层小人（头/身/手/道具）+ devControls 切换按钮。
问题：CSS 画精细角色一致性差、圆角化糊；且"小人当主体但没投入主体该有的逐帧动作可辨识度"。
彻底放弃。

**第二版 AgentRoom 静态小人 + 大道具**：深紫房间 + 悬浮小人 + 道具叙事。
问题：① 小人全 12 状态本体完全相同，只靠 emoji 气泡区分（AI 味）；② 单色蓝靛紫沉闷；
③ **大道具盖住小人**（碎纸机/档案柜压在身上）。属"道具挡主体"的结构性毛病。

**第三版 场景叙事 + rAF 插值**：小人按状态**横向飘到对应家具工位**，家具**自己运转**
表达动作。小人本体只悬浮，全程不渲染手臂。补上了"中断即转"的工程要求。
遗留问题：小人可辨识动作稀少（问答链路只剩几个静态站位），且演出层（道具/error）的
CSS 选择器写成全局裸类名 `.canvas`/`.dude`，命不中 CSS Module 的哈希类名——**实际是死代码**，
连线/放大镜/error 红光从不显示。

**第四版（当前）行为队列状态机**：把"stage → 单一落点"升级为"stage → 一段行为剧本"。
小人在 stage 内部**循环演出**（searching＝档案柜翻找→抱文件走回桌→翻阅→循环；writing＝
打字偶尔挠头/伸懒腰…），真实 idle 时**加权随机**做些闲事（喝咖啡/踱步/发呆/打瞌睡）。
同时把演出层选择器从死掉的全局裸类名改为「`[data-action]`/`[data-stage]` 属性选择 + 稳定
全局类（`.ar-dude`/`.p-doc`…）」，让道具与姿态真正显示。

### 8.3 当前机制（第四版：行为队列状态机）

**行为 = {目标工位 x, 微动作, 停留时长}**。`useAgentPosition` 逐个消费行为：先走到 x
（rAF 插值），到位后把微动作写到 canvas 的 `data-action` 上停留 ms，再取下一个；队列空则
续排——工作 stage 循环剧本、idle 加权随机（间隔 8~20s）。剧本数据在 `behaviors.ts`。

**微动作驱动姿态与随身道具**：`data-action` ∈ walk/carry/type/read/flip/rummage/gesture/
drink/daze/doze/scratch/stretch/idle-breathe。`roomScenes.css` 按 `[data-action]` 给小人
（`.ar-dude`）套对应姿态动画，并显隐随身道具（手持文件 `.p-doc`、瞌睡 zzz 气泡 `.p-zzz`）
与场地道具（连线 `.p-link`、放大镜 `.p-glass`）。

**横向工位带**：房间 5 个固定家具 = 5 个工位（电脑桌+显示器 ｜ 咖啡角 ｜ 打印机 ｜ 档案柜 ｜
销毁台）。家具环境动画（显示器明灭、档案柜抽屉、咖啡热气、扫描线）由家具自身 CSS 常驻表达。

**红线**：`data-action` 只是 **stage 内部的表现层编排**，stage 本身只来自真实 RunEvent；
瞌睡/闲逛只在真实 idle 发生，绝不显示虚假阶段文案。**可达性**：AgentRoom 只挂工作台，只会
收到问答链路的 idle/searching/checking/writing/linking/error；文档处理各 stage 仅
StyleGallery 预览目录可见（`sceneMap.ts` 已就近标注"仅预览可达"）。

**多彩配色**：橙卫衣(`--dude-body`) + 粉高光(`--dude-body-hi`)，让小人在深紫房间跳成焦点。
画布背景分层做"夜间小剧场"纵深（顶部柔光 + 底边暗角）。

### 8.4 文件分工

| 文件 | 职责 |
|---|---|
| `drawDude.ts` | 像素小人绘制：8×8 图案数组 + 编译器 → box-shadow 字符串。改图案/配色只动这里 |
| `behaviors.ts` | **行为剧本数据源**：工位坐标、工作 stage 循环剧本、idle 加权随机、预览首帧 |
| `useAgentPosition.ts` | **行为队列状态机**：rAF 位置插值 + data-action 微动作 + 中断即转（见 §8.5） |
| `sceneMap.ts` | stage → {label, detail, busy} 配置 + 可达性标注 |
| `AgentRoom.tsx` | 房间结构：家具带 + 道具层 + 小人(含随身道具) + 画布下方运行轨迹 |
| `AgentRoom.module.css` | 房间/家具/小人静态样式与悬浮基线（哈希类，内部自洽） |
| `roomScenes.css` | **演出层**：`[data-action]` 小人姿态 + 随身/场地道具 + `[data-stage=error]`。全局文件，一律用属性选择 + 全局类命中（不依赖哈希类名） |

### 8.5 中断即转（最关键的工程决策）

**问题**：stage 快速连续变化时（Agentic RAG 一次问答会反复 searching→linking→checking），
纯 CSS `transition` **无法读取动画中间帧的真实位置**作为新起点，会导致小人瞬移/跳变。
CSS keyframes 更糟——它和 transition 互斥，切换时必跳。

**解决**：`useAgentPosition.ts` 用 JS + `requestAnimationFrame` 手动逐帧插值：
1. `currentXRef` 每帧记录小人**真实 left**（含插值中间帧），用 ref 不触发 re-render
2. `stage` 变化 → useEffect → **丢弃当前行为队列**，以 `currentXRef`（真实当前位置）为起点
   重排新 stage 的剧本，rAF 逐帧走向队首行为目标
3. 中途 stage 再变 → effect cleanup 取消旧 rAF/timeout → 新剧本**从真实当前位置出发** = 中断即转
4. 走路时长按**距离比例**计算（`max(320ms, 距离×16)`），远近步速自然
5. 直接写 `style.left`（DOM）、`data-action`（DOM 属性），都不进 React state——60fps/高频
   进 state 会拖性能；React 只控 `data-stage`/`data-busy`，不碰 `data-action`（互不打架）

**idle 为什么现在能闲逛了**：第三版曾因 keyframes/transition 互斥而放弃 idle 循环。第四版
改用同一套 rAF 状态机排 idle 剧本（喝咖啡/踱步/发呆/瞌睡），和工作剧本共用"中断即转"逻辑，
真实事件一到立刻丢队列切走——闲逛与中断可靠性不再冲突。`prefers-reduced-motion` 时状态机
直接静立在落点仅呼吸、不排剧本（红线：不走动）。

### 8.6 box-shadow 像素法（小人怎么画的）

不用图片、不用 SVG，用 1 个 `<div>` + 多层 `box-shadow` 画整个小人：
- `drawDude.ts` 里写一个 8×8 的**字符串图案数组**（`.` 透明，其余字符映射颜色）
- `compile()` 把图案编译成一串 `"{x}px {y}px 0 0 {color}"` 投影坐标
- 渲染时 `style={{ boxShadow: DUDE_SHADOW }}`，硬边像素、性能好、易维护

这是"真栅格"做法（符合像素动画要求），改小人造型/配色**只改 PATTERN/COLOR 两个常量**。

## 9. 无障碍与性能

经 PR 审计整改（F1-F7 等），无障碍达到可公开展示/写进简历的硬指标：

- `prefers-reduced-motion`：**双层兜底**——global.css 全局关闭 CSS 动画，
  `useAgentPosition` 的 rAF 层单独检测，开启时小人**静立在落点仅呼吸、不排行为剧本**（不走动）
- `:focus-visible`：键盘聚焦显示焦点环，鼠标点击不显示
- 语义化：`aria-label`/`aria-current` 与真实指向一致；WorkbenchView 补 sr-only `<h1>`
  （global.css 提供 `.sr-only` utility）
- 键盘可达：GraphView 的 Cytoscape canvas 不可 Tab，在实体详情空状态补**可 Tab 的实体
  按钮列表**作为并行可达路径（F5）
- 焦点管理：设置面板打开移焦到关闭按钮，关闭还焦到触发按钮（F6）
- 触控目标：引用角标按钮 ≥28px（F3）
- 颜色对比度：成功色/subtle 色加深，正文与背景满足 WCAG AA（F4）
- TopBar 状态灯接 `/health/deps` 真实连通状态，绿/红/灰三态 + aria-label（F10）
- 性能：动画优先 `transform`/`opacity`（GPU 合成）；位置插值走 rAF 直接写 DOM 不进 state；
  AgentRoom 屏幕发光仅 `data-busy=1` 时跑（省 CPU）
- Cytoscape 体积大（~650KB），是 build 警告的来源，非阻塞

## 10. 改动指南

### 加一个新状态（假设后端新增 stage）
1. `types/runEvent.ts` 的 `Stage` 加枚举值（前后端同源，后端也要加）
2. `sceneMap.ts` 加 `{label, detail, busy}` 配置（若运行时不可达，就近标注"仅预览可达"）
3. `behaviors.ts`：`STAGE_HOME` 加歇脚坐标；若是可达工作 stage，在 `WORK_SCRIPTS` 加循环剧本
4. `AgentRoom.tsx` 若需新随身/场地道具，加一个全局类 `<div class="prop p-xxx">` 或小人子元素
5. `roomScenes.css` 加 `[data-action="新动作"] .p-xxx { ... }`（演出层一律用属性选择 + 全局类）
6. typecheck + build 验证；`?preview` 进 StyleGallery 看全 12 状态静态首帧

### 给小人加一个新微动作（stage 内部演出）
1. `behaviors.ts` 的 `AgentAction` 加动作名，在剧本里排上 `{x, action, ms}`
2. `roomScenes.css` 加 `[data-action="新动作"] .ar-dude { animation: ar-xxx ... }` + keyframes
3. 红线：只能是真实 stage 内部的表现，不得据此显示虚假阶段文案

### 改小人造型/配色
只动 `drawDude.ts` 的 `PATTERN`（图案）和 `COLOR`（颜色映射）。颜色建议改 `tokens.css`
的 `--dude-*` 变量，drawDude 里引用变量名。

### 会话功能切真实联调（已完成）
后端会话 API（`/api/conversations/*`）就绪后，前端从 mock 切真实的步骤（**已执行**，记录留查）：
1. `api/conversations.ts` 顶部 `USE_MOCK` 改 `false`（已改）
2. 删掉文件末尾整段 mock 数据 `_mockCitations` / `_mockStore` / `mockList` 等（已删）
3. 起后端联调：新建会话 → 提问（带 conversationId）→ 终态落答案 → 切换会话验历史回灌
4. typecheck + build；SSE 鉴权若配了 API_KEY 注意 L7（见 §9 注）

## GraphView 质量重构补充（2026-07）

GraphView 现采用 local-first：先加载社区概览，再按中心节点请求有界 subgraph；社区请求失败时回退到实体列表。已有节点位置会被复用，过滤和邻域展开不会触发整图随机重排。

`api/graph.ts` 映射 `documentId`、`confidence`、`evidenceChunkId`。关系详情展示方向、类型、置信度、证据 chunk 和文档；缺失证据会明确显示 Missing evidence。实体列表使用原生 button，保留键盘访问和焦点恢复。
