# 设计规格：AgentRoom 像素小人工位基地升级

> 2026-07-06 大脑窗口经 brainstorming 流程与用户确认方向后定稿。
> 本规格只覆盖前端 `AgentRoom` 的视觉与动作升级，不改后端事件协议。

## 背景与目标

当前 `AgentRoom` 已有稳定的 `RunEvent -> stage -> 行为` 主链路，但门面体验还有三个明显短板：

1. 小人五官辨识度不足，人物感偏弱，难以支撑“灵动的小工程师”人设。
2. `drink` 更像“走到咖啡角附近站住”，不是明确发生了交互。
3. `doze` 的困倦状态主要靠姿态和 `zzz` 气泡表达，眼睛与嘴部缺少可读细节。

目标：把当前深紫小房间升级为 **Agent 小基地**。保留像素语言、保留真实 `RunEvent`
驱动的红线，把小人做成更机灵的像素小工程师，并让咖啡角、主工位、资料柜形成更完整
的“会工作的房间”。

## 已确认决策

| 维度 | 结论 | 理由 |
|---|---|---|
| 改动级别 | 成套改版 | 用户明确要一起评估小人、房间、家具、咖啡角 |
| 世界观 | 保留像素元素 | 不改成非像素风，延续项目招牌组件 |
| 人物气质 | 灵动的小工程师 | 比当前更机灵，但仍有专业底色 |
| 房间方向 | 混合型小基地 | 兼具档案感与开发者工位感，更贴合 Archigraph |
| 配色决策 | 由大脑统一定 | 用户不单独拍板单色，而是接受成套搭配 |
| 后端参与 | 本批次默认不需要 | 当前问题集中在前端表现层，不涉及 `Stage` 契约变更 |

## 总体视觉方向

推荐方向：**Agent 小基地版**

- 房间不是纯游戏场景，也不是纯档案室，而是一个“会工作的智能工位基地”。
- 颜色以 **深靛 / 炭蓝** 为基底，维持夜间小剧场感。
- 系统反馈色用 **青蓝**，用于屏幕、扫描、连线、放大镜等“数字行为”。
- 生活感与聚焦光用 **琥珀暖光**，用于咖啡角、台灯、局部反射。
- 小人主体保留暖色倾向，但从“暖橙色块”升级成更精致的工程师配色，避免与房间割裂。

需要避免：

- 赛博朋克或游戏厅式高饱和霓虹，压过项目的专业感。
- 过度复古像素网页风，把房间做成纯怀旧展示。
- 每个家具都抢戏；画面焦点必须优先回到小人和当前工位。

## 小人升级设计

### 1. 人物定位

小人升级为 **像素小工程师**：

- 仍然是中性的小助手，不做强性别表达。
- 相比当前版本，更强调“机灵、会思考、会工作的陪伴感”。
- 保留眼镜作为辨识锚点，但眼镜不再只是装饰线条，而是人物身份的一部分。

### 2. 造型方向

- 保留 `box-shadow` 像素法，不切换到图片精灵或 SVG。
- 重画 `PATTERN`，让头部稍大、身体更利落，增强剪影辨识度。
- 发型、脸部、躯干的明暗层次更明确，避免“只有一坨暖色 body”。
- 眼镜框、眼睛、嘴部都进入“可读细节层”，尤其要支持不同动作状态切换。

### 3. 五官规则

- 常态：眼镜清晰、眼睛明确可见，整体表情轻松但专注。
- `doze`：眼睛切成两条横线，明显区别于常态。
- `doze`：嘴改为小圆口，配合轻微呼吸节奏。
- `daze` / `drink` / `read` 等状态可以只做姿态变化，不强求每个状态都换五官。

## 房间与家具升级设计

### 1. 空间定位

房间升级为 **会工作的 Agent 小基地**，保留当前布局逻辑，但让每个工位职责更清楚：

- `desk`：主工位，承担 `checking` / `writing` / 阅读回桌段。
- `coffee`：生活角，承担放松、补能量、短暂停顿。
- `cabinet`：资料与知识入口，承担 `searching`。
- `link`：关系推演与连线工位，承担 `linking`。
- 其他家具继续作为背景支撑，不新增独立复杂子系统。

### 2. 视觉层次

- 主工位、咖啡角、资料柜三者层级必须更清楚，不能“都差不多亮”。
- 主工位和咖啡角加一点暖光，让它们像真实可用的生活工位。
- 资料柜和连线台维持偏冷的系统反馈色，形成“知识工作”和“生活停顿”的对比。
- 房间整体比现在更像一个完整空间，而不是几件道具平铺在画布上。

### 3. 咖啡角

咖啡角是本次重点升级对象：

- 让小人从“到点站立”变成“取杯-喝-停顿”的完整交互。
- 咖啡角本体要有可感知的响应，例如杯位状态、蒸汽变化或被取用后的空位感。
- 交互的视觉焦点应来自“小人和咖啡之间发生了什么”，不是单独给咖啡加大动画。

## 动作编排升级

### `drink`

从单动作升级为短剧本：

1. 小人走到咖啡角。
2. 停顿一拍，表现“取杯”。
3. 杯子作为随身道具出现。
4. 小人后仰或抬手喝一口。
5. 小停顿后回到空闲基调。

实现要求：

- 允许在 `behaviors.ts` 中把 `drink` 拆成两到三个连续行为。
- 必须保持现有“中断即转”能力，新的 `stage` 到来时可打断该剧本。

### `doze`

从“歪头 + zzz”升级为“姿态 + 五官 + 呼吸”组合：

- 姿态仍保留轻微歪头。
- 眼睛切成横线，清楚可见。
- 嘴做小圆口。
- 呼吸节奏要慢，不可抢戏。
- `zzz` 保留，但成为辅助信息，不再是唯一信息源。

### 其他动作

- `idle-breathe` / `daze` 保持克制，避免全屋持续躁动。
- `searching` / `linking` / `writing` 的逻辑不推翻，只做视觉衔接优化。
- `prefers-reduced-motion` 规则保持原样：不让小人到处走动。

## 代码边界与文件影响

本批次只动前端，且集中在现有 `AgentRoom` 相关文件：

- `frontend/src/components/AgentRoom/drawDude.ts`

- `frontend/src/components/AgentRoom/AgentRoom.tsx`

- `frontend/src/components/AgentRoom/sceneMap.ts`

- `frontend/src/views/StyleGallery/StyleGallery.module.css`

原则：

- 不改 `RunEvent`、`Stage`、`useRunEvents` 契约。
- 不改后端 SSE / chat / graph 逻辑。
- 不新增依赖。
- 不把 `AgentRoom` 重写成 canvas 或 sprite sheet 方案。

## 验收标准

### 用户可感知验收

1. 小人第一眼更像“像素小工程师”，而不是抽象色块。
2. `drink` 明显是与咖啡的交互，不再只是站位。
3. `doze` 不看 `zzz` 也能看懂是在打瞌睡。
4. 房间更像一个完整的小基地，主工位、咖啡角、资料柜分工更清楚。

### 工程验收

1. `Stage` 数据源红线不被破坏，状态仍只来自真实 `RunEvent`。
2. `prefers-reduced-motion` 不退化。
3. `StyleGallery` 仍能预览各 stage，不出现空白或错位。
4. `npm run typecheck` 与 `npm run build` 通过。

## 后端配合判断

本批次 **不要求后端窗口配合**。

只有在前端工人实施时发现以下情况，才再单独派后端小清单：

- 需要新增 `Stage` 或重命名现有 stage；
- 需要后端额外传递动作级元数据；
- 需要改变 `RunEventTimeline` 的结构字段。

按当前设计，这三种情况都不应发生。
